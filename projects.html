myFlix React Case Study

{ Objective }

Project Duration: 3 months
Role: Web Developer
Link: https://myflix-app-react.netlify.app/
Tools Used: MERN Stack, Bootstrap, Redux,
Visual Studio Code, Github, Netifly, Postman

{ The Challenge }

To create a personal project from scratch, develop both back-end/front-end and connect the server-side to the client-side. The application was developed using the MERN stack. MERN stands for MongoDB, Express, React, Node, after the four key technologies that make up the stack. myflix is designed to work as an online movie library that will provide the users with data about movies, genres, and directors. Users will be able to create an account with information that includes a unique username, password, email, birthday, and a list of their favorite movies.

{ Purpose & Context }

The goal of this project was to have a demanding full-stack application that I could add to my professional portfolio. The problem that I wanted to solve was to build the complete server-side and client-side for the application from scratch. I wanted to showcase my skills in handling databases, and creating a React web application from start to finish. My role as the web developer was to create the back-end database, API, and front-end interface for the user to interact with.

{ Process & Approach }

The first step is to set up the back-end logic for creating, reading, updating, and deleting data that is stored on the server and then connect these to the front-end user interface. Once the back-end is complete, tested, and working as it should be then the front-end of the application is developed. The application should look great while not trying to reinvent the wheel. All the data that is stored on the server should be easily accessible to the user on the application.

[ Building the Server-side ]

I created a RESTful API using Node.js and Express, which interacts with a non-relational database (MongoDB). To access the API one would use commonly used HTTP methods such as GET, POST, PUT, or DELETE. The back-end is designed to contain two main collections: one for the user’s information, and another for the movies. Both of these collections contain data in JSON format. To avoid having the user enter their log-in information every time they refreshed the page or made a new selection, password hashing and a JWT token were added to the page. The token is valid for a few days which allows the user to revisit the page and not have to log in after every visit.

To test the API, I used Postman. Authentication and authorization for the user are also provided in the form of basic HTTP authentication and JWT authentication. Every time a user logs in, a token is generated and stored for future use.

View Documentation

[ Building the Client-side ]

Once the backend of the application was complete and everything was working how it should, I started to think about how a user would like the application to look and feel. This was my first approach to React; it is bundled with Parcel and styled with React bootstrap. To be able to access different routes directly from the web address input, React Router was added to the application to provide direct access to the different components, as well as a direct URL for each object in the collection.

There are several interfaces for the user to view such as the main movie view component which allows the user to search for a movie, read a brief description of each movie, and the ability to scroll through the list till they find something they like.

Within the main view, a user can then select to view more information about each movie. When a movie is selected the user will be brought to a single movie view component where they will see information about the movie’s description, genre, and director. The user can also select to add the movie to their favorites list if they wish to store it for future viewing.

The navigation bar allows users to view their profile information, go back to the main movie view, or log out of the application. Once in the profile view, users can choose to update their profile if they want to change their username or password. The profile page also holds the list of movies that the user had selected to add to their favorites list. They can simply click the movie poster and view all the information about the movie. If they wish to remove the movie they can select remove favorite and the movie will then be removed from their favorites.

{ Challenges }

This application was one of my favorites to complete during my web development course. Learning how to create the back-end API and being able to implement it with a working front-end application was very rewarding. During the development process, I became very familiar with VS Code and learned how to use the built-in terminal to quickly install new packages. While I did not run into many issues developing the back-end of the application that quickly changed when I started on the front-end. Jumping right into React and React Redux was a bit of a learning curve and I started to doubt my abilities in coding. With the help of my tutor and a few one on one sessions to figure out why certain logic was not working I was quickly able to pick it up and get over that doubt. Finishing this project was very rewarding and I am happy with how the end result turned out.

{ Duration }

This application was developed in two parts. The back-end of the application took me about 4 weeks to complete and make sure everything functioned as I had intended. The front-end of the application took a little longer and was finished in 8 weeks. Not knowing exactly what to expect from React and how to use it correctly was a huge learning curve, and I got stuck at several points during the development of the application. After completing the project I can say I learned a lot about using React and how to face different errors that may arise in the development process.
